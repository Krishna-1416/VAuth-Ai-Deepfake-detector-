# Gemma 4 Good Hackathon: Implementation Plan

## 🏆 Hackathon Strategy
**Target Tracks:**
1. **Safety & Trust ($10,000):** Building a framework for transparency by moving from "black-box" deepfake detection to explainable, grounded AI forensics.
2. **Unsloth Special Technology Track ($10,000):** Showcasing the best fine-tuned Gemma 4 model using Unsloth for a specific, impactful task (Explainable Deepfake Analysis).

## 🎯 The "Wow" Factor
Standard detection tools return a simple probability (e.g., "85% Fake"). Our solution leverages Gemma 4's multimodal and reasoning capabilities to act as an **AI Forensic Analyst**. It will not only detect the manipulation but *explain why* it made that decision (e.g., "The lighting on the subject's face does not match the background environment", "Unnatural blurring detected around the jawline").

---

## 🛠️ Execution Phases

### Phase 1: Fine-Tuning Gemma 4 (Colab & Unsloth)
**Objective:** Fine-tune Gemma 4 to analyze image frames and output structured forensic reasoning.
* **Script:** `notebooks/finetune_gemma4_unsloth.ipynb`
* **Workflow:**
  1. Load a custom dataset via CSV containing `image_path`, `label` (Real/Fake), and `explanation` (detailed forensic reasoning).
  2. Use `unsloth` to optimize memory and speed for Gemma 4 fine-tuning on a Colab T4/L4 GPU.
  3. Format the prompts for multimodal analysis: `<image> Analyze this image for deepfake artifacts. Return JSON with 'score' and 'reasoning'.`
  4. Export the LoRA adapters and upload the fine-tuned model to a Hugging Face Space for backend inference.

### Phase 2: Backend - Smart Video Processing
**Objective:** Optimize video analysis to feed Gemma 4 the most relevant data.
* **File:** `backend/detectors/video_detector.py`
* **Changes:**
  * Replace the naive sequential 1fps frame extraction.
  * Implement **Randomized Smart Sampling**: Randomly pick N frames, or use lightweight OpenCV heuristics to pick frames with high variance/motion to ensure diverse analysis.

### Phase 3: Backend - Gemma 4 Inference Integration
**Objective:** Upgrade the Hugging Face API integration from a standard classifier to a generative multimodal endpoint.
* **Files:** `backend/detectors/image_detector.py`, `backend/detectors/video_detector.py`
* **Changes:**
  * Update `API_URL` to point to the new Gemma 4 Hugging Face Space endpoint.
  * Rewrite the API call payload to send the image + text prompt instead of just the image byte stream.
  * Add JSON parsing logic to extract both the `score` (probability) and the `explanation` generated by Gemma 4.

### Phase 4: Output & Result Aggregation
**Objective:** Surface Gemma 4's reasoning to the user interface.
* **File:** `backend/utils/result_builder.py`, `backend/main.py`
* **Changes:**
  * Adjust the `build_result` function to accept and attach the Gemma-generated textual explanation.
  * Ensure the Fast API `/detect` endpoint correctly maps this explanation so the React frontend can display the AI's forensic report directly in the UI.
